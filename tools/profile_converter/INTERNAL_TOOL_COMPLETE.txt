Redesigned the Profile Converter GUI as an internal tool for the
Pandaforge project with the following improvements:

✅ AUTOMATIC PATH CONFIGURATION
   • No browsing for directories
   • Uses OrcaSlicer-main/resources/profiles (in project)
   • Outputs to BambuStudio-2.5.0.66/resources/profiles (in project)
   • Zero configuration needed

✅ HIERARCHICAL SELECTION
   • Tree view with brands and printer models
   • Select entire brands (all printers)
   • Select individual printers from any brand
   • Mixed selection (brands + individual models)
   • Visual indicators (📁 brands, 🖨️ printers, ✓ Klipper)

✅ SMART SELECTION OPTIONS
   • "Select All Brands" - Convert everything
   • "Select All Klipper" - Convert only Klipper printers
   • "Deselect All" - Clear selection
   • "Expand All" / "Collapse All" - Show/hide models
   • Filter box - Search brands in real-time

✅ ENHANCED UI
   • Model count per brand
   • Klipper count per brand
   • Visual selection highlighting
   • Real-time selection summary
   • Expandable/collapsible tree

🔧 TECHNICAL IMPLEMENTATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Architecture:
  • PrinterModel class - Represents individual printer
  • PrinterBrand class - Represents brand with models
  • ProfileConverterGUI class - Main application

Selection State:
  • Set of tuples: (brand_name, model_name)
  • Brand selection: (brand_name, None)
  • Model selection: (brand_name, model_name)
  • Prevents conflicting selections

Tree Structure:
  • ttk.Treeview with hierarchical data
  • Brands as parent nodes
  • Models as child nodes
  • Tags for styling (selected, klipper)

Path Detection:
  • Uses Path(__file__).parent.parent.parent
  • Automatically finds project root
  • Sets OrcaSlicer and BambuStudio paths
