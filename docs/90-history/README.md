# History

Historical documentation lives here.

## What Belongs Here

- superseded plans
- early agent/context docs
- rename notes tied to older branch states
- dated investigations, verification writeups, and implementation reports

## How To Use It

- Start with [`../../INDEX.md`](../../INDEX.md) and [`../01-start-here/current-state.md`](../01-start-here/current-state.md).
- For build work, read [`../03-task-guides/build.md`](../03-task-guides/build.md) first and only then open `plans/build-guide-v1.md` for the longer backstory and lessons.
- Only open archived docs when you need historical intent or branch archaeology.
- Do not treat archived docs as the current build/layout truth without checking the live source tree.

## Reports

`reports/` contains point-in-time snapshots such as investigations, test reports, and implementation summaries. They are useful for history, but not as the default agent entry point.
